进程 ：系统分配资源的最小单位，主要是分配内存。

不同进程间的数据不共享，都是独立的空间，同一进程中的不同线程资源共享。

进程数据共享和可采用队列的方式实现。



线程 ：是cpu调度的最小单位，线程之间数据共享，要数据安全性要加锁操作。

python中GIL 全局解释器锁：无法利用多核，同一时间内，只允许一个线程操作cpu。

a线程操作完CPU 会将锁给 b线程  然后b才会去执行

所以 python多线程 不适用于cpu密集型场景。

io密集型场景 可使用多线程 节省时间。



协程：存在于线程中的最小单位，主要为了解决异步io。

在a协程挂起状态下，会去激活b协程工作。节省等待阻塞时间，提高效率。

实现原理 是基于python中生成器执行原理，当遇到yield会进去阻塞状态。__send__会激活。

协程对象会保存上下文信息
